## 1. Definición de la problemática

| Tema | Explicación |
|-----|-------------|
| Contexto general | Registrar información en el expediente clínico electrónico es una de las tareas administrativas que más tiempo consume durante una consulta médica. Aunque muchas instituciones ya cuentan con sistemas digitales, estos no siempre están diseñados para adaptarse al ritmo real de trabajo de los médicos. |
| Tiempo dedicado a capturar información | Durante cada consulta el médico debe llenar múltiples campos en el sistema, buscar diagnósticos dentro de listas extensas y escribir la nota clínica. En algunos casos, encontrar el diagnóstico correcto puede tomar más tiempo que redactar la propia nota médica. |
| Uso de abreviaturas | Los médicos suelen utilizar abreviaturas para escribir más rápido. Sin embargo, muchas de estas abreviaturas no siguen un estándar común y pueden tener significados distintos dependiendo de la especialidad médica, lo que puede causar confusión cuando otro profesional revisa el expediente. |
| Omisiones en la exploración física | Las normas médicas indican que la exploración física debe registrarse de forma completa. En la práctica, algunos médicos anotan únicamente lo relacionado con su especialidad y omiten otras partes del cuerpo. Si el expediente se revisa posteriormente, estas omisiones pueden generar problemas legales. |
| Limitaciones tecnológicas | En algunos hospitales los sistemas informáticos pueden volverse lentos o dejar de funcionar cuando muchos usuarios los utilizan al mismo tiempo. Esto dificulta acceder a la información del paciente cuando se necesita y puede retrasar la atención médica. |
| Clasificación de urgencias | En muchos servicios de urgencias la prioridad de atención se decide manualmente por el personal de admisión. Esto puede generar inconformidad entre los pacientes, ya que no siempre comprenden por qué otras personas son atendidas primero. |

## 2. Público objetivo e impacto esperado

| Perfil de usuario | Situación actual | Cómo ayuda el sistema |
|------------------|------------------|-----------------------|
| Médicos generales y especialistas | Dedican una parte importante de su jornada a capturar información en el sistema. También utilizan abreviaturas que pueden resultar confusas para otros profesionales. | El sistema busca reducir el tiempo dedicado a la captura de datos y ayudar a interpretar abreviaturas comunes para que el expediente sea más claro. |
| Personal de enfermería | Generalmente registran signos vitales y datos iniciales del paciente, pero muchas veces lo hacen en papel o en sistemas separados del expediente clínico. | El sistema permite registrar esta información directamente en el expediente desde el inicio del proceso de atención. |
| Personal administrativo | Registran los datos personales del paciente en sistemas distintos al expediente clínico, lo que puede generar duplicación de información. | El sistema integra los datos administrativos con el expediente médico para evitar capturas repetidas. |
| Clínicas y hospitales | Los sistemas utilizados en distintas unidades médicas no siempre se comunican entre sí, lo que dificulta consultar el historial de un paciente. | El sistema está diseñado para facilitar el intercambio de información entre diferentes unidades médicas. |
| Servicios de urgencias | La prioridad de atención depende del criterio del personal que recibe al paciente, lo que puede generar conflictos o percepciones de trato injusto. | El sistema puede apoyar el proceso de clasificación de urgencias con criterios claros y visibles para el personal médico. |

## 3. Propuesta de valor

| Elemento | Explicación |
|----------|-------------|
| Captura de información más sencilla | El sistema permite que el médico registre información de forma similar a como lo hace normalmente, sin tener que aprender procesos complicados o cambiar completamente su forma de trabajar. |
| Reducción de omisiones en el expediente | Antes de cerrar el expediente, el sistema puede indicar si falta información importante. Esto ayuda a que el registro médico sea más completo. |
| Apoyo en la clasificación de urgencias | El sistema puede ayudar a organizar la atención en los servicios de urgencias utilizando criterios médicos claros para definir prioridades. |
| Control de acceso a la información | Cada miembro del equipo de salud accede únicamente a la información necesaria para realizar su trabajo, lo que ayuda a proteger la privacidad del paciente. |
| Preparación para compartir información entre instituciones | El sistema utiliza una estructura de datos que facilita que la información del paciente pueda compartirse entre distintos hospitales o clínicas cuando sea necesario. |

## 4. Diferencias clave con los sistemas existentes

| Aspecto | Situación común en sistemas actuales | Propuesta del sistema |
|---------|--------------------------------------|-----------------------|
| Uso de abreviaturas | Las abreviaturas se registran tal como el médico las escribe, lo que puede generar confusión. | El sistema puede reconocer abreviaturas comunes y ayudar a interpretarlas correctamente. |
| Búsqueda de diagnósticos | El médico debe buscar manualmente en listas largas de diagnósticos. | El sistema puede sugerir diagnósticos relacionados con la información registrada previamente. |
| Formularios | Los formularios suelen ser rígidos y no cambian según el tipo de consulta. | El sistema puede sugerir información adicional dependiendo de los datos ya capturados. |
| Registro previo a la consulta | En muchos lugares los datos iniciales se anotan primero en papel. | Los datos pueden registrarse directamente en el sistema desde el inicio de la atención. |
| Clasificación de urgencias | La prioridad de atención se define manualmente. | El sistema puede apoyar el proceso utilizando criterios médicos claros. |
| Comunicación entre hospitales | Los sistemas suelen estar separados y no comparten información fácilmente. | El sistema busca facilitar el intercambio de información entre diferentes unidades médicas. |
| Control de acceso | Los permisos de acceso no siempre están claramente definidos. | El sistema establece roles claros para cada tipo de usuario. |
| Exploración física | Algunas partes del expediente pueden quedar incompletas. | El sistema puede recordar al médico completar información importante antes de cerrar el expediente. |

## 5. Posicionamiento respecto a sistemas privados y sistemas institucionales

| Tipo de sistema | Características |
|-----------------|----------------|
| Sistemas privados | Ofrecen soporte técnico y herramientas avanzadas, pero suelen tener costos constantes de uso y no siempre se adaptan a las necesidades del sistema de salud público. |
| Sistemas institucionales | No generan costos directos para los usuarios, pero en algunos casos presentan limitaciones técnicas o dificultades para actualizarse. |
| Sistema propuesto | Busca combinar las ventajas de ambos modelos: ser accesible, adaptable al contexto médico local y diseñado para facilitar el trabajo cotidiano del personal de salud. ||
